Overview

Mullvad VPN Tray is a lightweight Linux system tray applet that puts Mullvad VPN exit-node management directly in your desktop.

What It Does

This tool integrates with Tailscale's exit-node feature. Instead of using the Tailscale CLI or web dashboard, you can:

  • Select Mullvad exit nodes from a categorized menu
  • Open a web-based server picker
  • Set up automatic rotation (change server every X minutes)
  • Lock rotation to a specific country
  • Connect to a random server with one click

Installation

1. Install System Dependencies

# Debian / Ubuntu
sudo apt update
sudo apt install python3-gi gir1.2-ayatanaappindicator3-0.1 gir1.2-webkit2-4.1

# Verify WebKit support
python3 -c "from gi.repository import WebKit2; print('WebKit2 OK')"

# Verify Indicator support
python3 -c "from gi.repository import AyatanaAppIndicator3; print('Indicator OK')"

2. Install and Configure Tailscale

# Install Tailscale
curl -fsSL https://tailscale.com/install.sh | sh

# Login (creates account if needed)
tailscale up

# Verify you're connected
tailscale status
Mullvad VPN exit nodes appear in tailscale status only when your Tailscale network has the Mullvad relay enabled. Check your Tailscale admin console if no nodes appear.

3. Install Mullvad VPN Tray

# Clone the repository
git clone https://github.com/eroslifestyle/mullvad-vpn-tray.git
cd mullvad-vpn-tray

# Make executable
chmod +x vpn-tray.py

# Run it
./vpn-tray.py

4. Run at Startup (Optional)

# Create systemd user service
mkdir -p ~/.config/systemd/user

# Create the service file
cat > ~/.config/systemd/user/vpn-tray.service << 'EOF'
[Unit]
Description=Mullvad VPN Tray
After=network-online.target
Wants=network-online.target

[Service]
Type=notify
ExecStart=/path/to/vpn-tray.py
Restart=on-failure
WatchdogSec=30

[Install]
WantedBy=default.target
EOF

# Enable and start
systemctl --user daemon-reload
systemctl --user enable --now vpn-tray.service

# Check status
systemctl --user status vpn-tray.service
The WatchdogSec=30 directive tells systemd to monitor the service. Mullvad VPN Tray sends WATCHDOG=1 via NOTIFY_SOCKET every few seconds. If systemd doesn't receive it for 30 seconds, it restarts the service.

Connection Modes

Every connection mode ultimately runs tailscale set --exit-node=<IP>. The UI just makes it easier.

🌐 Web Picker

Access: Menu → "Pick a server"

Opens an embedded HTML popup with:

  • Dropdown to filter by Country
  • Dropdown to filter by City
  • Search box for quick text filtering
  • Clickable server list grouped by location

Click any server to connect instantly.

🎲 Fast Connect

Access: Menu → "Fast connect"

Picks a random exit node from any country and connects immediately.

This is the quickest way to change your exit location.

🌍 Country Submenu

Access: Menu → [Country flags]

Each country with Mullvad servers appears as a submenu item.

Clicking a country picks a random node within that country.

Example: Click 🇩🇪 Germany → connects to a random German server.

🔄 Rotate Now

Access: Menu → "Rotate now"

Forces an immediate rotation:

Country Lock Behavior
OFF Random node from any country
ON Random node in current country

❌ Disconnect

Access: Menu → "Disconnect"

Runs tailscale set --exit-node= (empty).

Your traffic no longer exits via Mullvad. It uses Tailscale's default route.

Country Lock

When Country Lock is ON, rotation stays within the current country:

# Country Lock ON + in Italy
Rotation → Random Italian server (Milan, Rome, Palermo...)

# Country Lock OFF
Rotation → Random server anywhere (US, DE, SE, JP...)

Configuration File

Settings persist in ~/.config/vpn-tray/config.json:

{
  "interval_min": 0,        // 0 = disabled
  "country_lock": false     // true = stay in current country
}
You can edit this file directly, or use the tray menu. Changes take effect immediately.

Systemd Service

Running as a systemd user service gives you watchdog protection and automatic startup on login.

Full Service File

[Unit]
Description=Mullvad VPN Tray
Documentation=https://github.com/eroslifestyle/mullvad-vpn-tray
After=network-online.target
Wants=network-online.target
PartOf=graphical-session.target

[Service]
Type=notify
ExecStart=/opt/vpn-tray/vpn-tray.py
Restart=on-failure
RestartSec=5
WatchdogSec=30
Environment=DISPLAY=:0
Environment=XDG_CURRENT_DESKTOP=Unity

[Install]
WantedBy=default.target

Troubleshooting

Tray icon not showing

1. Check if you have a system tray available:

# Verify GTK and Indicator are installed
python3 -c "from gi.repository import Gtk; print('GTK OK')"
python3 -c "from gi.repository import AyatanaAppIndicator3; print('OK')"

2. Try the fallback indicator:

# Some environments need the older AppIndicator3
python3 -c "from gi.repository import AppIndicator3; print('OK')"

3. Check if your desktop environment supports indicators (GNOME, XFCE, LXQt work well; GNOME Shell may need an extension).

No exit nodes found

1. Verify Tailscale is running and connected:

tailscale status

2. Check for ExitNodeOption peers:

tailscale status --json | jq '.Peer | to_entries[] | select(.value.ExitNodeOption == true)'

3. If empty, enable Mullvad in your Tailscale admin console under Access Controls or relay configuration.

Rotation not working

1. Enable logging and run manually:

python3 vpn-tray.py 2>&1 | grep vpn-tray

2. Check the interval setting (must not be 0).

3. Verify country_lock is set as intended.

Service won't start

1. Check the logs:

journalctl --user -u vpn-tray -e

2. Verify the ExecStart path is correct.

3. Ensure DISPLAY is set if running in a graphical session:

Environment=DISPLAY=:0
Connection is slow

1. Try a server closer to your physical location.

2. Use the web picker to compare latency.

3. Some countries have faster Mullvad relays than others.

Country lock doesn't work

The country lock persists per session. It reads the current exit node's country at the time you enable it. If you want to change the locked country:

  1. Connect to a server in your desired country
  2. Enable Country Lock
  3. Rotation will now stay within that country

Architecture

Data Layer (Module Functions)

Function Purpose
sd_notify(state) Sends WATCHDOG=1 to systemd via NOTIFY_SOCKET
load_config() / save_config(cfg) JSON persistence in ~/.config/vpn-tray/
ts_status_peers() Queries tailscale status --json for ExitNodeOption peers
list_nodes() Filters and formats exit nodes (country, city, IP, priority)
current_node() Returns the active exit node, if any
set_node(ip) / clear_node() Runs tailscale set --exit-node=

UI Layer (VpnTray Class)

Method Purpose
__init__ Creates Indicator, loads config, starts polling timer
build_menu() Constructs the GTK menu from the node list
_chooser_html() Generates HTML for the web picker popup
on_choose_node() Shows the WebKit picker window
_on_web_connect(ip) JavaScript callback — connects to selected server
refresh() Updates tray icon and status text
on_random() / on_random_fast() Random connect (country-aware / global)
on_rotate() / apply_rotation() Rotation logic with country-lock support
set_interval(minutes) Sets/resets the GLib rotation timer
